# Manuel client simplifie

## Objectif

Cette application permet de visualiser les parcelles, de consulter leurs informations, et d'utiliser une intelligence artificielle comme aide a la decision.

Le parcours utilisateur est accompagne par des didacticiels guides integres a l'interface.

## Parcours d'utilisation base sur les didacticiels

## 1) Presentation generale de l'interface

Ce premier didacticiel montre les zones principales :

- la carte des parcelles ;
- la zone de recherche et de filtrage ;
- la zone des priorites de labellisation ;
- la zone des graphiques ;
- la zone des parametres du modele ;
- la zone de demandes d'amelioration ;
- la zone d'aide interactive.

Objectif : comprendre rapidement ou cliquer selon le besoin metier.

## 2) Labellisation via les priorites

Ce didacticiel explique comment traiter les parcelles prioritaires :

1. Ouvrir la liste prioritaire.
2. Choisir une parcelle a traiter.
3. Lire le diagnostic de l'intelligence artificielle.
4. Corriger manuellement la classe si necessaire.
5. Valider ensuite le travail humainement.

Objectif : accelerer la correction tout en conservant un controle humain final.

## 3) Lecture des graphiques d'analyse

Ce didacticiel accompagne la lecture des graphiques :

- repartition des surfaces par classe ;
- evolution des classes dans le temps ;
- comparaison visuelle entre classes.

Objectif : expliquer simplement les tendances et evolutions observees.

## 4) Lecture detaillee d'une parcelle

Ce didacticiel montre comment analyser une parcelle en detail :

1. Ouvrir la fiche parcelle.
2. Lire les informations clefs.
3. Examiner l'imagerie disponible.
4. Changer le mode de visualisation pour comparer.
5. Lire le diagnostic de l'intelligence artificielle.

Objectif : prendre une decision locale sur une parcelle precise.

## 5) Envoi d'une remarque ou d'une demande

Ce didacticiel montre comment signaler un besoin :

1. Ouvrir la zone de demandes.
2. Choisir le type de demande.
3. Decrire clairement le contexte et le resultat attendu.
4. Envoyer la demande pour traitement.

Objectif : faciliter les echanges entre utilisateurs et equipe technique.

## 6) Comprendre les analyses du modele

Ce didacticiel presente la partie suivi du modele :

- indicateurs principaux de performance ;
- frequence de mise a jour ;
- evolution des performances ;
- tableau de confusion ;
- rapport par classe ;
- referentiel des classes utilisees.

Objectif : donner une vision claire de la qualite du modele dans le temps.

## Presentation simple de l'intelligence artificielle (partie serveur)

L'intelligence artificielle fonctionne cote serveur, en arriere-plan.
Elle prepare des resultats lisibles pour l'utilisateur, sans remplacer la decision metier.

## Ce qu'elle apporte

- priorisation des parcelles a traiter ;
- lecture rapide des tendances globales ;
- appui a l'analyse parcelle par parcelle ;
- suivi regulier de la qualite du modele.

## Ce qu'elle ne remplace pas

- l'expertise terrain ;
- la verification humaine ;
- la validation finale par les responsables.

## Bonnes pratiques

- verifier la qualite des donnees saisies ;
- utiliser les didacticiels pour suivre le bon parcours ;
- croiser les propositions de l'intelligence artificielle avec l'expertise metier ;
- conserver une validation humaine avant toute decision finale.

## Assistance

En cas de difficulte :

1. Noter l'action realisee.
2. Noter le message affiche.
3. Transmettre ces informations a l'equipe d'assistance.
